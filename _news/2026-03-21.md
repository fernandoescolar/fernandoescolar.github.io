## Kubernetes: Ingress2Gateway 1.0
Kubernetes publicó la versión 1.0 de Ingress2Gateway, una herramienta diseñada para facilitar la migración desde controladores Ingress tradicionales hacia la Gateway API. El artículo explica objetivos prácticos: ofrecer rutas de migración seguras, mapear comportamientos comunes de Ingress y proporcionar utilidades que ayudan a automatizar la conversión y reducir riesgos durante la transición. La publicación llega en el contexto del retiro previsto de Ingress‑NGINX en marzo de 2026 y destaca guías y recomendaciones para planificar migraciones en entornos de producción. [Fuente: Kubernetes Blog](https://kubernetes.io/blog/2026/03/20/ingress2gateway-1-0-release/). ([kubernetes.io](https://kubernetes.io/blog/2026/03/20/ingress2gateway-1-0-release/))

## Kubernetes: Ejecutando agentes en Kubernetes con Agent Sandbox
Kubernetes detalla un enfoque para ejecutar agentes (sistemas agent‑based/agentic AI) dentro de clústeres usando un proyecto llamado Agent Sandbox. El post describe arquitectura, casos de uso para agentes que requieren estado y acceso a recursos persistentes, y prácticas para aislar, probar y supervisar agentes en producción. También se discuten implicaciones de seguridad y operativas al pasar de llamadas stateless a arquitecturas centradas en agentes. [Fuente: Kubernetes Blog](https://kubernetes.io/blog/2026/03/20/running-agents-on-kubernetes-with-agent-sandbox/). ([kubernetes.io](https://kubernetes.io/blog/2026/03/20/running-agents-on-kubernetes-with-agent-sandbox/))

## .NET: Acelerando .NET MAUI con agentes de IA
El blog oficial de .NET publicó un artículo sobre cómo agentes de IA personalizados están ayudando al flujo de contribución y desarrollo de .NET MAUI. El texto (guest post) describe un conjunto de "skills" y agentes —pr-review, write-tests-agent, sandbox-agent y otros— que automatizan reproducción de issues, generación de tests, intentos de corrección y verificación automática, reduciendo tiempos de resolución y mejorando cobertura de pruebas. Se incluyen ejemplos prácticos de uso y recomendaciones para integrarlos en el proceso de contribución. [Fuente: .NET Blog](https://devblogs.microsoft.com/dotnet/accelerating-dotnet-maui-with-ai-agents/). ([devblogs.microsoft.com](https://devblogs.microsoft.com/dotnet/accelerating-dotnet-maui-with-ai-agents/))

## Google Cloud y NVIDIA: ampliando infraestructura AI en GTC

## GKE: Inference Gateway multiclúster para servir modelos a escala
Google Cloud anunció la vista previa de Multi‑cluster GKE Inference Gateway, una extensión del Gateway API orientada a servir modelos de ML a escala entre múltiples clústeres y regiones. La solución presenta recursos CRD como `InferencePool` e `InferenceObjective` para agrupar backends con GPUs/TPUs y políticas de enrutamiento basadas en métricas del modelo (p. ej. uso de cache KV o in‑flight requests). El objetivo es mejorar disponibilidad, balanceo y aprovechamiento de aceleradores en despliegues globales de inferencia. [Fuente: Google Cloud Blog](https://cloud.google.com/blog/products/containers-kubernetes/multi-cluster-gke-inference-gateway-helps-scale-ai-workloads). ([cloud.google.com](https://cloud.google.com/blog/products/containers-kubernetes/multi-cluster-gke-inference-gateway-helps-scale-ai-workloads))
