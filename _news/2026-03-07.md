## OpenAI lanza GPT‑5.3 Instant y GPT‑5.4 Thinking/Pro
OpenAI publicó una actualización de su línea GPT‑5 con al menos tres variantes: GPT‑5.3 "Instant" orientado a respuestas rápidas y conversacionales, GPT‑5.4 "Thinking" optimizado para razonamiento multi‑paso y flujos de trabajo complejos, y GPT‑5.4 Pro para cargas de trabajo profesionales de mayor exigencia. La nota de lanzamiento destaca trade‑offs entre latencia y capacidad (Instant vs Thinking/Pro), mejoras en manejo de contextos largos y en workflows con herramientas. Estas familias apuntan tanto a usuarios de ChatGPT como a desarrolladores vía API; en paralelo la página de release notes de ChatGPT recoge la llegada de GPT‑5.4 Thinking a la experiencia de producto. [Fuente](https://help.openai.com/en/articles/6825453-chatgpt-release-notes%23.eot). ([help.openai.com](https://help.openai.com/en/articles/6825453-chatgpt-release-notes%23.eot?utm_source=openai))

## Microsoft Foundry: REST API v1 GA, SDKs y herramientas para agentes
Microsoft Foundry publicó un resumen de novedades donde el REST API v1 llega a GA, y los SDKs (incluido un beta para .NET) están alineándose sobre esa superficie estable. Además se anuncian integraciones de modelos (Claude Opus, Sonnet, Grok), mejoras en catálogo de modelos y en el `Model Router`, y herramientas de desarrollo como el AI Toolkit para VS Code (Agent Inspector, Tool Catalog, Agent Builder). También se menciona Foundry Local (capacidad de ejecutar modelos multimodales en entornos desconectados/sovereign). Esto implica que desarrolladores que usan Azure Foundry disponen de APIs productivas y SDKs en pre‑release que migrarán pronto a GA. [Fuente](https://devblogs.microsoft.com/foundry/whats-new-in-microsoft-foundry-feb-2026). ([devblogs.microsoft.com](https://devblogs.microsoft.com/foundry/whats-new-in-microsoft-foundry-feb-2026?utm_source=openai))

## MCP C# SDK v1.0: SDK oficial para Model Context Protocol (.NET)
El equipo de .NET publicó el lanzamiento de la versión 1.0 del SDK C# para el Model Context Protocol (MCP). La entrega incorpora mejoras en descubrimiento de servidores de autorización, flujos de autorización enriquecidos, metadata ampliada y patrones para llamadas a herramientas y manejo de peticiones de larga duración. Es una noticia relevante para desarrolladores C# que construyen integraciones con infraestructuras de modelos y agentes, ya que facilita la adopción de patrones de tool‑calling y manejo de contextos en .NET. [Fuente](https://devblogs.microsoft.com/dotnet/release-v10-of-the-official-mcp-csharp-sdk/). ([devblogs.microsoft.com](https://devblogs.microsoft.com/dotnet/release-v10-of-the-official-mcp-csharp-sdk/?utm_source=openai))

## Go publica releases de seguridad y parches (1.26.1 / 1.25.8)
El equipo de Go anunció releases puntuales de seguridad: Go 1.26.1 y Go 1.25.8 incluyen varias correcciones de seguridad, entre ellas el CVE‑2026‑27137 relacionado con la verificación de restricciones de email en `crypto/x509` (comportamiento incorrecto al aplicar constraints en ciertos certificados). La entrada de golang‑announce recomienda actualizar a las versiones parcheadas; la issue pública y el changelog técnico en GitHub detallan las revisiones. Si tu infraestructura (servicios, NAS, contenedores) usa Go 1.26/1.25, planifica la actualización y revisa dependencias. [Fuente](https://groups.google.com/g/golang-announce/c/EdhZqrQ98hk). ([groups.google.com](https://groups.google.com/g/golang-announce/c/EdhZqrQ98hk))

## AKS: Dynamic Resource Allocation y soporte para NVIDIA vGPU en AKS
El blog de AKS publicó guía y anuncio sobre Dynamic Resource Allocation (DRA) para aceleradores, mostrando integración con virtualización de GPU (NVIDIA vGPU) y cómo aprovechar `DeviceClasses`/`ResourceClaims` en Kubernetes para asignación dinámica de aceleradores. El post describe patrones de despliegue para cargas AI/ML en AKS, beneficios de vGPU (multi‑tenant, perfiles previsibles) y recomendaciones para orquestación en clústeres que comparten GPUs físicas. Interesante tanto para equipos cloud como para quienes diseñan infra local/homelab con Kubernetes y GPUs. [Fuente](https://blog.aks.azure.com/). ([blog.aks.azure.com](https://blog.aks.azure.com/?utm_source=openai))

## CoPaw (AgentScope): estación de agentes AI local open‑source gana tracción
CoPaw (Co Personal Agent Workstation), proyecto ligado a AgentScope, aparece como una plataforma open‑source para ejecutar agentes personales localmente: interfaz web local, integración con múltiples canales (Discord, Telegram, etc.), y soporte para modelos locales via `llama.cpp`, Ollama o backends cloud. La web oficial y repositorio indican instalaciones vía `pip` o contenedor Docker y promueven ejecución local para privacidad y control de datos. Es un ejemplo notable del movimiento "local first" para agentes y tooling de RAG/agentic workflows. [Fuente](https://copaw.bot/). ([copaw.bot](https://copaw.bot/?utm_source=openai))

## QNAP retira una versión de firmware (QTS) tras problemas; revisar actualizaciones
La comunidad y foros de QNAP reportaron que una build de QTS fue liberada y posteriormente retirada debido a problemas (firmware release withdrawn). Si gestionas QNAP NAS, revisa notificaciones oficiales de QNAP, evita instalaciones automáticas si no son necesarias y sigue las instrucciones de su soporte para mitigar riesgos. En entornos homelab es buen momento para comprobar backups, snapshots y políticas de actualización automatizada. [Fuente](https://community.qnap.com/t/qts-5-2-9-3410-build-20260214-released-withdrawn/5472). ([community.qnap.com](https://community.qnap.com/t/qts-5-2-9-3410-build-20260214-released-withdrawn/5472?utm_source=openai))
