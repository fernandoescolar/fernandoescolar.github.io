## Azure Databricks Lakebase aterriza en marzo
Microsoft anunció en el MWC 2026 que Azure Databricks Lakebase llegará en marzo con disponibilidad administrada para operadores de telecomunicaciones.  
La solución combina lakehouse y base de datos transaccional gestionada para sincronizar datos de red y negocio sin duplicación.  
El servicio ofrece clonación instantánea y escala bajo demanda gracias a la separación de almacenamiento y cómputo.  
Lakebase se apoya en Microsoft Fabric como capa unificada de gobernanza y seguridad para acelerar los despliegues de IA en telecom.  
Los equipos de datos deben preparar migraciones desde almacenes heredados y planes de cumplimiento antes de la apertura general. ## Google amplía su infraestructura con nuevo centro de datos en Texas
Google confirmó una nueva instalación en el condado de Wilbarger (Texas) con acuerdos de energía limpia para minimizar el impacto en la red local.  
La inversión forma parte de un plan de gasto de capital de hasta 185 000 millones de dólares en 2026 para infraestructura de IA y nube.  
El sitio empleará sistemas de refrigeración por aire avanzados para eliminar el uso operativo de agua en climatización.  
El proyecto viene acompañado de contratos energéticos con AES y compromisos de más de 7 800 MW de generación renovable.  
La compañía proyecta que estas expansiones sostendrán el crecimiento del backlog de Google Cloud, que ya supera los 240 000 millones de dólares. ([techbuzz.ai](https://www.techbuzz.ai/articles/google-bets-big-on-texas-with-new-data-center-expansion?utm_source=openai))

## Azure MCP Server ahora habla Python
El equipo de Azure SDK lanzó soporte nativo de Python para Azure MCP Server mediante paquetes en PyPI y ejecución con uvx.  
La novedad permite a desarrolladores de agentes aprovechar más de 40 integraciones de servicios Azure desde entornos Python sin depender de Node.js o .NET.  
La distribución incluye plantillas de configuración para IDEs como VS Code, Visual Studio, IntelliJ y clientes MCP compatibles.  
Se recomienda usar uvx para ejecutar el servidor bajo demanda y mantenerlo siempre actualizado sin instalaciones persistentes.  
La integración con GitHub Copilot SDK facilita orquestar flujos agentic con autenticación Azure y herramientas declarativas en JSON. ([devblogs.microsoft.com](https://devblogs.microsoft.com/azure-sdk/azure-mcp-server-better-python-support?utm_source=openai))  
https://devblogs.microsoft.com/azure-sdk/azure-mcp-server-better-python-support

## Parche de febrero para .NET corrige CVE-2026-21218
Microsoft publicó los paquetes .NET 10.0.3, 9.0.13 y 8.0.24 para mitigar la vulnerabilidad de bypass de seguridad CVE-2026-21218 en ASP.NET Core.  
Las actualizaciones incluyen instaladores, contenedores y paquetes Linux para mantener sincronizados entornos de desarrollo y producción.  
Los equipos deben revisar las notas de problemas conocidos antes de desplegar en clusters o bundles de hosting IIS.  
La publicación confirma que .NET Framework no recibe nuevos parches este mes, aunque se mantienen las guías de soporte.  
Se recomienda planificar la adopción inmediata para evitar exposiciones en servicios web y workloads cloud. ([devblogs.microsoft.com](https://devblogs.microsoft.com/dotnet/dotnet-and-dotnet-framework-february-2026-servicing-updates/))

## Docker Desktop 4.62.0 cierra fallo en grpcfuse
Docker liberó la versión 4.62.0 con corrección a la vulnerabilidad CVE-2026-2664 en el módulo grpcfuse utilizado por Windows, Linux y macOS.  
La actualización renueva el kernel Linux interno a la versión 6.12.69, reduciendo riesgos de incompatibilidad con virtualización reciente.  
Se añadieron perfiles para Docker MCP Toolkit que organizan servidores MCP en catálogos personalizados para los equipos.  
La entrega soluciona un error que ignoraba la desactivación de comprobaciones automáticas de actualización en segundo plano.  
Usuarios de Intel Mac obtienen soporte para vLLM Metal dentro de Docker Model Runner para acelerar inferencias locales. ([docs.docker.com](https://docs.docker.com/desktop/release-notes/))  
https://docs.docker.com/desktop/release-notes/

## Alibaba Cloud retirará Ingress-NGINX gestionado
Alibaba Cloud anunció que su servicio ACK dejará de mantener el add-on Nginx Ingress Controller el 30 de noviembre de 2026.  
La decisión responde al retiro del proyecto comunitario kubernetes/ingress-nginx tras el periodo de mantenimiento hasta marzo de 2026.  
La compañía insta a los usuarios a planificar migraciones a controladores alternativos o configuraciones propias con SLA.  
Se anticipa que, una vez retirado, no habrá nuevas versiones ni parches de seguridad para el complemento gestionado.  
Los clientes deberán validar dependencias de red y políticas de seguridad antes de la fecha límite para evitar interrupciones. ([alibabacloud.com](https://www.alibabacloud.com/en/notice/product_changecontainer_service_for_kubernetes_end_of_maintenance_for_nginx_ingress_controller_6a6?_p_lc=1&utm_source=openai))

## Intel archiva proyectos open source en Go
Intel comunicó la descontinuación de cuatro proyectos open source escritos en Go orientados a sus aceleradores de datos.  
Entre los repositorios archivados figura IXL-GO, biblioteca que exponía capacidades de Intel IAA y DSA para compresión y CRC acelerados.  
La empresa justifica la medida por la baja adopción de estas herramientas en la comunidad Go.  
Los desarrolladores deberán evaluar alternativas o mantener forks propios si dependen de estos aceleradores.  
No se anticipan reemplazos oficiales, lo que deja a los usuarios con responsabilidad sobre futuras correcciones. ([phoronix.com](https://www.phoronix.com/news/Intel-Stops-Go-Projects?utm_source=openai))
