## Windows lleva la IA local a las apps Electron
Microsoft publicó una guía práctica para integrar IA generativa en Electron usando el paquete experimental `@microsoft/windows-ai-electron`, que habilita modelos locales como Phi Silica en Copilot+ PC sin escribir código nativo adicional. ([devblogs.microsoft.com](https://devblogs.microsoft.com/ifdef-windows/use-windows-on-device-ai-in-your-electron-app?utm_source=openai))
El ejemplo “Electron on Windows Gallery” incluye plantillas de generación de texto, resumen y OCR listas para reutilizar en proyectos comerciales. ([devblogs.microsoft.com](https://devblogs.microsoft.com/ifdef-windows/use-windows-on-device-ai-in-your-electron-app?utm_source=openai))
La configuración se apoya en el Windows App Development CLI y añade la capacidad `systemAIModels` al manifiesto para autorizar el acceso a NPU. ([devblogs.microsoft.com](https://devblogs.microsoft.com/ifdef-windows/use-windows-on-device-ai-in-your-electron-app?utm_source=openai))
El artículo detalla cómo comprobar la disponibilidad del modelo con `LanguageModel.EnsureReadyAsync()` y pautas para empaquetar la app con identidad de Windows. ([devblogs.microsoft.com](https://devblogs.microsoft.com/ifdef-windows/use-windows-on-device-ai-in-your-electron-app?utm_source=openai))
Microsoft invita a la comunidad a aportar comentarios sobre el SDK para priorizar nuevas API de IA en JavaScript. ([devblogs.microsoft.com](https://devblogs.microsoft.com/ifdef-windows/use-windows-on-device-ai-in-your-electron-app?utm_source=openai))
Fuente: ([devblogs.microsoft.com](https://devblogs.microsoft.com/ifdef-windows/use-windows-on-device-ai-in-your-electron-app?utm_source=openai))
