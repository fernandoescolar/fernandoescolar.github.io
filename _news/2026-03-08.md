## Release v1.0 del SDK MCP en C# (MCP C# SDK)
Microsoft publicó el lanzamiento v1.0 del SDK oficial en C# para Model Context Protocol (MCP). La entrada del equipo .NET describe soporte completo para la especificación 2025-11-25, mejoras en descubrimiento y autorización, soporte para elicitation por URL, herramientas para llamadas en sampling, y mecanismos mejorados para peticiones de larga duración y tareas duraderas. El artículo incluye ejemplos de uso, opciones de configuración y recomendaciones de retención para streams de eventos, orientado a desarrolladores .NET que quieran construir servidores o clientes MCP. ([devblogs.microsoft.com](https://devblogs.microsoft.com/dotnet/release-v10-of-the-official-mcp-csharp-sdk/?utm_source=openai))
[Fuente](https://devblogs.microsoft.com/dotnet/release-v10-of-the-official-mcp-csharp-sdk/)

JetBrains publicó la documentación y recursos de la versión de ReSharper integrada para Visual Studio Code (versión 2026.1), describiendo configuración de estilo, inspecciones, y opciones de registro y licencia. ## VS Code 1.110: mejoras para agentes en desarrollo
El equipo de Visual Studio Code publicó detalles de la versión 1.110 centrada en hacer los agentes prácticos para tareas de desarrollo reales. Entre las novedades están memoria de sesión con compactación manual, forking de conversaciones, plugins/skills para agentes, herramientas experimentales de navegador integradas para que agentes inspeccionen y validen UI, y hooks para automatizar comprobaciones (por ejemplo, lint/test antes de aplicar cambios). La nota enfatiza control, observabilidad y flujos más seguros para usar agentes en flujos de trabajo de desarrollo. ([code.visualstudio.com](https://code.visualstudio.com/blogs/2026/03/05/making-agents-practical-for-real-world-development?utm_source=openai))
[Fuente](https://code.visualstudio.com/blogs/2026/03/05/making-agents-practical-for-real-world-development)

## Windows Terminal Preview 1.25
Microsoft lanzó Windows Terminal Preview 1.25 con búsqueda en la página de ajustes, un editor UI para acciones y keybindings, soporte del protocolo de teclado Kitty, y mejoras de rendimiento de I/O. La versión añade además una página de Settings dedicada a extensiones y mejoras en la persistencia de layouts; está disponible vía GitHub/WinGet y como paquete Preview en la Tienda. Es una actualización útil para desarrolladores que usan terminales y agentes en flujos interactivos. ([devblogs.microsoft.com](https://devblogs.microsoft.com/commandline/windows-terminal-preview-1-25-release?utm_source=openai))
[Fuente](https://devblogs.microsoft.com/commandline/windows-terminal-preview-1-25-release)

## GPT-5.4 disponible en Microsoft Foundry
Microsoft anunció la disponibilidad de GPT‑5.4 en Microsoft Foundry, describiéndolo como un modelo orientado a ejecutar y completar tareas en entornos de producción con mejoras en razonamiento sostenido, uso integrado de “computer use” (capacidades de interacción con herramientas/archivos y ejecución controlada), y variantes Pro para análisis más profundos. Microsoft destaca controles operativos de Foundry para gobernanza y despliegue empresarial, y publica detalles de precios y disponibilidad inicial. Esta actualización refuerza el enfoque en agentes confiables y workflows de IA en la nube empresarial. ## ExpressVPN lanza servidor MCP beta
ExpressVPN puso en beta un servidor MCP que permite a herramientas de IA interactuar directamente con el cliente VPN (p. ej. comprobar estado, cambiar servidor, o solucionar conexiones) mediante el Model Context Protocol. El movimiento es relevante porque expone control de funciones del sistema (VPN) a flujos agenticos estandarizados; los equipos de seguridad y desarrolladores deberán evaluar riesgos y controles de autorización antes de integrarlo en entornos de producción. ([tomsguide.com](https://www.tomsguide.com/computing/vpns/expressvpn-launches-its-mcp-server-that-lets-ai-tools-control-vpn-connections-and-its-an-industry-first?utm_source=openai))
