## Sonnet 5 (Anthropic) ya disponible en la Agent Platform de Google Cloud
Anthropic ha añadido Sonnet 5 al catálogo de modelos accesibles desde la Agent Platform de Google Cloud, ofreciendo a las organizaciones una opción adicional para cargas de trabajo agenticas empresariales. Según la nota recopilatoria de Google Cloud, Sonnet 5 se presenta como reemplazo directo de Sonnet 4.6 en la plataforma, con mejoras en razonamiento, generación de código y capacidades para interactuar con entornos de escritorio y navegador. La disponibilidad en la Agent Platform facilita a los equipos integrar Sonnet 5 en arquitecturas multi-agente y en flujos de trabajo gestionados dentro del ecosistema de Google Cloud. Esta incorporación forma parte de una serie de actualizaciones recientes de la plataforma para facilitar despliegues de modelos y operaciones ML a escala. [Fuente: What's new with Google Cloud (Jul 2, 2026)](https://cloud.google.com/blog/topics/inside-google-cloud/whats-new-google-cloud). ([cloud.google.com](https://cloud.google.com/blog/topics/inside-google-cloud/whats-new-google-cloud))

## Claude en Microsoft Foundry: disponibilidad general en Azure
Microsoft anunció que Claude (modelos de Anthropic) está disponible en Microsoft Foundry y alojado en Azure, permitiendo a clientes empresariales usar Claude dentro del entorno de controles, facturación y gobernanza de Azure. El comunicado subraya que Claude en Foundry está orientado a cargas agenticas y tareas complejas como generación y refactorización de código, y que la integración aprovecha infraestructura acelerada por NVIDIA Blackwell Ultra. Microsoft destaca la posibilidad de enmarcar el uso del modelo dentro de las políticas y RBAC de Azure, y ofrece rutas de despliegue que conectan Foundry Agent Service con los controles empresariales ya existentes. Esta disponibilidad facilita a las organizaciones trasladar pilotos de agentes a entornos de producción con garantías de administración y gobernanza. [Fuente: Azure Blog — Claude in Microsoft Foundry (Jun 29 / Jul 1, 2026)](https://azure.microsoft.com/en-us/blog/claude-in-microsoft-foundry-is-now-generally-available/). ([azure.microsoft.com](https://azure.microsoft.com/en-us/blog/claude-in-microsoft-foundry-is-now-generally-available/?utm_source=openai))

## DigitalOcean DOKS: Node Readiness Controller en public preview
DigitalOcean anunció en sus notas que DOKS incorpora un Node Readiness Controller en vista previa pública; este controlador mantiene un taint `NoSchedule` en nodos hasta que las condiciones necesarias (por ejemplo drivers GPU y componentes críticos) reporten estado saludable, evitando que pods se programen en nodos no listos. Para clústeres con pools GPU esto ayuda a prevenir errores al programar cargas que requieren aceleradores y permite personalizar métricas de salud que gobiernan la elegibilidad de scheduling sin redeployar componentes. La funcionalidad llega integrada y gestionada por DOKS, facilitando su adopción en entornos que usan GPUs para ML/inferencia. [Fuente: DigitalOcean Docs — Kubernetes (Changelog / 2 Jul 2026)](https://docs.digitalocean.com/products/kubernetes/). ([docs.digitalocean.com](https://docs.digitalocean.com/products/kubernetes/?utm_source=openai))

## WSL Containers: Microsoft publica vista previa pública
Microsoft puso en pública vista previa la funcionalidad "WSL container" que añade `wslc.exe` y una API para ejecutar y gestionar contenedores Linux directamente desde WSL en Windows, evitando la necesidad de herramientas de terceros para desarrolladores. La preview incluye una CLI para flujos de trabajo de contenedores, integración con Intune/MDM, y ejemplos de uso con GPU (CUDA) y escritorios Linux en contenedores; Microsoft recomienda instalar la versión pre-release de WSL (`wsl --update --pre-release`) para probarla. La iniciativa representa un movimiento importante en la experiencia de desarrollo en Windows, integrando contenedores en la plataforma WSL y ofreciendo un camino nativo que puede reducir la dependencia de Docker Desktop en estaciones de trabajo. [Fuente: Windows Command Line Blog — WSL container public preview (Jun 29, 2026)](https://devblogs.microsoft.com/commandline/wsl-container-is-now-available-for-public-preview/). ([devblogs.microsoft.com](https://devblogs.microsoft.com/commandline/wsl-container-is-now-available-for-public-preview/?utm_source=openai))

## Google Cloud: Run:ai Model Streamer acelera carga de modelos en TPUs para GKE
Google Cloud destacó la integración del Model Streamer (de Run:ai) con TPUs utilizando Google Cloud Storage para acelerar el cold start de grandes modelos en GKE; la solución permite "streaming" de tensores a memoria y reduce mucho el tiempo de carga y el uso pico de RAM en hosts. Según la descripción técnica, en pruebas de cargas muy grandes la técnica duplicó la velocidad de carga y redujo memoria pico en un 50% al evitar cuellos de botella de disco y estrategias de doble-buffering. Esto es relevante para equipos que despliegan modelos en infraestructuras mixtas (GKE + TPUs) y buscan optimizar costes e incrementar aprovechamiento del hardware. [Fuente: What's new with Google Cloud (Jun 29 - Jul 3, 2026)](https://cloud.google.com/blog/topics/inside-google-cloud/whats-new-google-cloud). ([cloud.google.com](https://cloud.google.com/blog/topics/inside-google-cloud/whats-new-google-cloud))

## Google Cloud: AI Telemetry Collector (OpenTelemetry) para TPUs
En la misma recopilación de novedades, Google Cloud anunció un nuevo AI Telemetry Collector basado en OpenTelemetry pensado para estandarizar la monitorización de cargas TPU; el agente recopila métricas operativas útiles (memoria, latencia de red, utilización de núcleos) y las exporta a Cloud Monitoring, Prometheus o Grafana. La herramienta pretende ayudar a detectar fallos silenciosos y ajustar el despliegue de infraestructuras de entrenamiento/inferencia sin añadir carga CPU significativa, contribuyendo a reducir tiempos muertos de TPU y mejorar observabilidad en pipelines ML a gran escala. [Fuente: What's new with Google Cloud (Jun 29 - Jul 3, 2026)](https://cloud.google.com/blog/topics/inside-google-cloud/whats-new-google-cloud). ([cloud.google.com](https://cloud.google.com/blog/topics/inside-google-cloud/whats-new-google-cloud))
