## Telerik corrige CVE-2026-2878 en ASP.NET AJAX
Progress publicó la versión 2026.1.225 de Telerik UI for ASP.NET AJAX con un parche que corrige la vulnerabilidad CVE-2026-2878 en RadAsyncUpload, reforzando la entropía de los identificadores temporales.  
El paquete también canaliza todos los scripts externos de jQuery a través de un nuevo controlador que controla el orden de carga y las fuentes autorizadas.  
Componentes críticos como Gantt, HtmlChart y Spreadsheet dejan atrás referencias a objetos indefinidos que bloqueaban escenarios productivos tras la actualización 2026 Q1.  
La nueva RadSmartPasteButton gana soporte en el diseñador de Visual Studio y para RenderMode, acercando la funcionalidad a los demás controles modernos.  
El lanzamiento forma parte del ciclo de mantenimiento acelerado que Progress anunció tras varias incidencias de seguridad en 2025.

## SharePoint Framework marca la hoja de ruta previa a su 25.º aniversario
Microsoft adelantó que el nuevo toolbar de depuración para SharePoint Online terminará de desplegarse en febrero sin exigir actualización de paquetes SPFx.  
La versión 1.23, prevista para finales de marzo, abrirá plantillas y tooling de SPFx, permitiendo que partners personalicen los scaffolds corporativos.  
Para abril, la 1.23.1 pondrá en vista previa un CLI desacoplado que reemplaza al generador de Yeoman y permitirá overrides en paneles de listas modernas.  
La roadmap confirma un ciclo trimestral estable y compromisos continuos con la comunidad a través del hackathon SharePoint 2026, activo del 2 al 16 de marzo.  
Microsoft insiste en que la evolución de SPFx es clave para experiencias Copilot y SharePoint Embedded con énfasis en IA y extensibilidad segura.

## Microsoft refuerza Azure Local para entornos soberanos
Azure Local estrena modo desconectado que replica la experiencia de portal y gobernanza de Azure sin depender de conectividad con la nube pública.  
Microsoft 365 Local amplía el paquete offline con Exchange, SharePoint y Skype for Business soportados hasta 2035 dentro de nubes privadas soberanas.

Douglas Phillips subrayó que la expansión responde a requisitos de reguladores europeos y latinoamericanos que exigen control absoluto de datos y telemetría.  
La oferta incluye servicios de despliegue y actualización gestionada para que las organizaciones mantengan paridad funcional con Azure global sin perder aislamiento.

## Google abre su documentación a los agentes MCP
Google presentó el Developer Knowledge API en vista previa pública, proporcionando un endpoint MCP oficial que entrega fragmentos y páginas completas de documentación.  
El servicio permite a asistentes y herramientas basadas en LLM recuperar la guía vigente de productos como Firestore o Cloud Run evitando alucinaciones por APIs obsoletas.  
Los equipos habilitan la API con una clave de Google Cloud y configuran conectores MCP para búsquedas y descargas controladas desde la consola.  
Google planea añadir esquemas estructurados (p. ej. snippets de código y referencias) y reducir la latencia de indexación antes de la disponibilidad general.  
Con este lanzamiento, los tres grandes hyperscalers ya ofrecen servidores MCP: AWS y Microsoft lo hacen sin autenticación, mientras Google opta por API key y métricas.

## SoftServe lanza una suite agentic para el SDLC
SoftServe anunció la Agentic Engineering Suite, un marco que combina agentes de IA para modernizar y construir software reduciendo hasta un 90 % la carga manual.  
La suite se organiza en dos pilares: modernización y desarrollo, ambos orquestados mediante un núcleo agentic compatible con MCP para conectar herramientas preferidas.  
Incluye agentes especializados (QA, BA, Code Generation, Conversion, Architect, Maintenance, CI/CD) que automatizan pruebas, migraciones y pipelines end-to-end.  
El enfoque modular permite desplegar agentes como procesos CLI o APIs en nubes, on-prem o entornos híbridos, lo que facilita su adopción corporativa.  
SoftServe posiciona la oferta como respuesta a backlog masivo, lanzamientos greenfield y exigencias de gobernanza de IA en sectores regulados.

## Cilium 1.19 endurece la seguridad en clústeres masivos
La versión 1.19 de Cilium celebra diez años reforzando la malla eBPF con modos estrictos de IPsec y WireGuard que bloquean tráfico inter-nodo sin cifrar.  
El nuevo beta de Ztunnel aporta autenticación mutua transparente sin sidecars, alineando el plano de datos con expectativas de mallas de servicio modernas.  
Cambios en las políticas multiclúster restringen por defecto el tráfico a nodos locales si no se especifica origen, reduciendo exposiciones involuntarias.  
Multi Pool IPAM llega a estado estable y admite túneles IPv6, simplificando la asignación de direcciones en despliegues híbridos y multi-tenant.  
Hubble añade filtrado por estado de cifrado y etiquetas de políticas en eventos de drop, facilitando la observabilidad en incidentes complejos.

## Plural añade runtime de agentes y parches automáticos
Plural presentó Agent Runtime, un plano de control para definir agentes con permisos diferenciados según entorno, modelo y alcance operativo.  
Sobre este runtime debutan el Cluster Upgrade Agent y el módulo de remediación de CVE, pensados para automatizar upgrades de fleets Kubernetes y cierres de vulnerabilidades.  
La compañía apunta a reducir la fricción entre seguridad y plataforma, evitando que un único agente tenga acceso simultáneo a Terraform y cargas sensibles.  
La arquitectura permite encadenar agentes especializados y registrar sus acciones, facilitando auditoría y segregación de funciones.  
Plural promete extender el catálogo de agentes y publicar guías de integración con herramientas observabilidad en las próximas semanas.

## Nueva guía para resolver el error “package is not in GOROOT”
TechYorker publicó un manual actualizado que explica por qué Go muestra “package is not in GOROOT” cuando falla la resolución de módulos.  
El artículo desglosa la diferencia entre GOROOT y GOPATH, la influencia de GO111MODULE y cómo validar la configuración con `go env`.  
Propone flujos paso a paso para proyectos basados en módulos y en GOPATH, incluyendo el uso correcto de `go mod tidy` y `replace`.  
También aborda causas frecuentes como rutas con mayúsculas, auto-importaciones equivocadas del IDE y conflictos entre vendoring y módulos.  
Se incluyen recomendaciones para restablecer caches de herramientas como gopls y para entornos con múltiples versiones de Go instaladas.
