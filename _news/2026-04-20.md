## NET y ASPNET
Microsoft publicó su resumen de mantenimiento y parches para .NET y .NET Framework correspondientes a abril de 2026, que recopila correcciones de seguridad y otras actualizaciones de servicio para varias versiones. El post resume CVE atendidos y señala las ramas afectadas, de modo que equipos que administran infraestructura .NET (incluyendo imágenes de contenedores y servidores Windows) deberían revisar los detalles de los parches y planificar la aplicación según su ciclo de mantenimiento.  
Fuente: https://devblogs.microsoft.com/dotnet/dotnet-and-dotnet-framework-april-2026-servicing-updates

## Vista previa: .NET 11 Preview 3
El equipo de .NET publicó .NET 11 Preview 3, un nuevo avance de la plataforma que reúne cambios en runtime, SDK, bibliotecas y componentes web como ASP.NET Core. Entre las mejoras reportadas están novedades en el SDK, actualizaciones para ASP.NET Core (incluyendo soporte para nuevas opciones de compresión), y correcciones y optimizaciones en runtime y EF Core; como siempre, se trata de una preview destinada a pruebas y feedback, no a producción sin evaluación previa. Los equipos de desarrollo pueden probar la preview para validar compatibilidad y nuevas características en su ciclo de desarrollo.  
Fuente: https://devblogs.microsoft.com/dotnet/dotnet-11-preview-3

## azd: actualización unificada del Azure Developer CLI
Azure presentó el comando `azd update` para el Azure Developer CLI, que permite actualizar `azd` con un único comando independientemente del método original de instalación (winget, Homebrew, script, etc.). La novedad simplifica la vida del desarrollador al eliminar la necesidad de recordar el gestor de paquetes usado y ofrece además canales (stable/daily) para cambiar la línea de actualizaciones. Es una mejora práctica para flujos de desarrollo que dependen de `azd`.  
Fuente: https://devblogs.microsoft.com/azure-sdk/azd-update/

## A2UI v0.9: estándar de UI generativa para agentes
Google publicó detalles sobre A2UI v0.9, una especificación orientada a permitir que agentes generativos describan y rendericen UI usando bibliotecas y componentes existentes del cliente. A2UI busca ser agnóstico a frameworks y facilitar que agentes (locales o remotos) generen interfaces coherentes con el sistema de diseño de una aplicación, mejorando la integración entre agentes inteligentes y aplicaciones reales. Para equipos que exploran agentes que manipulan o generan interfaces, A2UI aporta un esquema para interoperabilidad y patrones de streaming.  
Fuente: https://developers.googleblog.com/a2ui-v0-9-generative-ui/

## LLM‑Wiki de Andrej Karpathy gana tracción como alternativa a RAG
La idea divulgada por Andrej Karpathy —llamada "LLM‑Wiki"— propone que, en lugar de depender solo de RAG (recuperación + generación) cada vez que se pregunta al modelo, el LLM compile y mantenga un wiki persistente (Markdown) que acumule y estructure conocimiento. Desde su gist original la propuesta se ha propagado rápidamente: hay forks, implementaciones y proyectos que exploran la arquitectura práctica (ingest → compilar wiki → consultar). Para equipos que trabajan con memoria de agentes y RAG, LLM‑Wiki plantea un enfoque de memoria acumulativa que merece evaluación.  
Fuente (gist original): https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f

## Audio Flamingo Next (NVIDIA): avances en modelos audio‑lenguaje
NVIDIA publicó el trabajo técnico de "Audio Flamingo Next", una familia de modelos diseñados para comprensión y razonamiento sobre entradas de audio de larga duración (reportes y checkpoints disponibles en repositorios/archivos académicos). El proyecto ofrece variantes orientadas a QA, transcripción y razonamiento temporal sobre audio extenso, y los artefactos están accesibles en Hugging Face y como preprints. Este tipo de modelos amplía las capacidades de LLMs multimodales al dominio audio y puede interesar a desarrolladores que integran análisis de voz, subtitulado y agentes conversacionales basados en audio.  
Fuente: https://arxiv.org/abs/2604.10905
