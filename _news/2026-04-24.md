## OpenAI lanza GPT‑5.5
OpenAI ha anunciado GPT‑5.5, descrito como “una nueva clase de inteligencia para trabajo real”. El modelo apunta a mejorar tareas de razonamiento multi‑paso, uso de herramientas y flujos de trabajo agenticos (codificación, análisis de datos, automatización), manteniendo latencias similares a versiones anteriores. OpenAI publica además una ficha técnica de seguridad y programas asociados (por ejemplo, un bug bounty para riesgos bio). [Fuente: OpenAI]. ([openai.com](https://openai.com/ta-IN/index/introducing-gpt-5-5/?utm_source=openai))

## OpenAI presenta Workspace Agents en ChatGPT
OpenAI ha puesto en preview las “workspace agents” para planes Business/Enterprise/Edu/Teachers: agentes compartidos en la nube que pueden ejecutar flujos de trabajo largos, acceder a sistemas autorizados (Slack, Gmail, etc.) y correr aun cuando los usuarios estén offline. La apuesta es convertir prácticas repetibles en agentes con gobernanza administrativa y controles de auditoría; la función estará gratuita en preview hasta una fecha anunciada. [Fuente: OpenAI]. ([openai.com](https://openai.com/index/introducing-workspace-agents-in-chatgpt/?utm_source=openai))

## Anthropic: memoria persistente para Claude Managed Agents (beta)
Anthropic ha publicado memoria persistente para sus Managed Agents —un almacenamiento en forma de “memory stores” que permite a agentes conservar aprendizajes entre sesiones, versionar y auditar cambios, y controlar accesos y retenciones. La oferta está orientada a casos empresariales donde la continuidad y trazabilidad de contexto mejora la estabilidad y reduce coste de inferencia. [Fuente: documentación Anthropic]. ([platform.claude.com](https://platform.claude.com/docs/en/managed-agents/memory?utm_source=openai))

## Google presenta TPU 8t (entrenamiento) y TPU 8i (inferencia)
En Google Cloud Next se anunciaron los TPUs de octava generación: un diseño bifurcado con TPU 8t optimizado para entrenamiento a gran escala y TPU 8i optimizado para inferencia de baja latencia. Google explica que la separación responde a las necesidades del “agentic era” (agentes que requieren inferencia masiva y baja latencia) y promete mejoras importantes en eficiencia y escalado. [Fuente: blog de Google]. ([blog.google](https://blog.google/innovation-and-ai/infrastructure-and-cloud/google-cloud/eighth-generation-tpu-agentic-era/))

El mensaje central fue pasar de conversación a agentes productivos integrados en la nube y en aplicaciones empresariales. [Fuente: cobertura Cloud Next]. ([itpro.com](https://www.itpro.com/cloud/live/google-cloud-next-2026-all-the-live-updates-as-they-happen?utm_source=openai))

## Microsoft publica parche de emergencia para ASP.NET Core (CVE‑2026‑40372)
Microsoft emitió un update fuera de banda para corregir una regresión en .NET que afectó a Data Protection de ASP.NET Core (vulnerabilidad registrada como CVE‑2026‑40372). La falla permitía la falsificación de ciertos artefactos protegidos (por ejemplo cookies/estado) en escenarios afectados; la recomendación incluye actualizar a la versión corregida, redeplegar, y considerar rotación de claves/invalidación de tokens si hubo exposición durante la ventana vulnerable. [Fuente: cobertura técnica]. ([arstechnica.com](https://arstechnica.com/security/2026/04/microsoft-issues-emergency-update-for-macos-and-linux-asp-net-threat/?utm_source=openai))

## Kubernetes 1.36 se publica con 70 mejoras
El proyecto Kubernetes ha publicado la versión v1.36, con alrededor de 70 mejoras entre grados Alpha/Beta/GA. Entre las novedades destacadas están la autorización fina del API de kubelet (Kubelet API Authorization) y nuevas capacidades orientadas a cargas AI/ML (mejor visibilidad de recursos, soporte para salud de recursos dinámicos y primeras APIs para Workload Aware Scheduling). Se recomienda revisar notas de versión antes de actualizar clusters de producción. [Fuente: blog oficial Kubernetes]. ([kubernetes.io](https://kubernetes.io/blog/2026/04/22/kubernetes-v1-36-release/?utm_source=openai))

## Repositorio Docker de Checkmarx KICS comprometido: imágenes y extensiones maliciosas
Investigadores detectaron que el repositorio oficial `checkmarx/kics` en Docker Hub fue comprometido y que se publicaron imágenes troceadas (tags sobrescritos y una etiqueta falsa v2.1.21) que contenían código para exfiltrar credenciales. También se identificaron extensiones de VS Code/Open VSX maliciosas relacionadas. Se aconseja asumir compromiso de secretos si se usaron esas imágenes/extensions, rotar credenciales, reconstruir desde fuentes verificadas y fijar imágenes por digest (SHA). [Fuente: BleepingComputer / Socket report]. ([bleepingcomputer.com](https://www.bleepingcomputer.com/news/security/new-checkmarx-supply-chain-breach-affects-kics-analysis-tool/?utm_source=openai))

## GitHub Copilot: mejoras en PR, sesiones agent y BYOK en VS Code

Cambios operativos en métricas y en la experiencia de agentes también fueron anunciados. [Fuente: recopilación de changelogs y notas]. ([jls42.org](https://www.jls42.org/en/news/ia-actualites-23-apr-2026?utm_source=openai))

## MINIX lanza T4000 y T5000: mini‑workstations para inferencia LLM local
MINIX presentó las mini‑workstations T4000 y T5000 orientadas a cargas generativas locales, basadas en módulos NVIDIA Jetson AGX Thor (arquitectura Blackwell). Los equipos se posicionan para inferencia de modelos de ~7B–70B parámetros en local, con opciones de memoria, conectividad 10GbE y soporte de la pila NVIDIA (JetPack, CUDA, TensorRT). Son una opción interesante para despliegues on‑prem/homelab que busquen inferencia privada y baja latencia. [Fuente: nota de producto / prensa]. ([benchlife.info](https://benchlife.info/minix-launches-t4000-t5000-generative-ai-mini-workstation-next-gen-on-premise-ai-computing/?utm_source=openai))
