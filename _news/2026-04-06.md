## Microsoft Agent Framework 1.0
Microsoft ha anunciado la llegada de la versión 1.0 de su Agent Framework para .NET y Python, marcando el SDK como listo para producción con API estables y compromiso de soporte a largo plazo. La plataforma unifica ideas de Semantic Kernel y AutoGen, e incluye conectores a múltiples proveedores de modelos, memoria modular (Redis, Neo4j, y otros), orquestación multi-agente y soporte para definiciones declarativas en YAML. La 1.0 incorpora además características para depuración y observabilidad de ejecuciones de agentes, y promueve migraciones desde Semantic Kernel o AutoGen. Fuente: [Microsoft DevBlogs — Agent Framework v1.0](https://devblogs.microsoft.com/agent-framework/microsoft-agent-framework-version-1-0/). ([devblogs.microsoft.com](https://devblogs.microsoft.com/agent-framework/microsoft-agent-framework-version-1-0/?utm_source=openai))

## Agent Governance Toolkit: seguridad en tiempo de ejecución para agentes
Microsoft publicó un conjunto de herramientas de gobernanza de agentes (Agent Governance Toolkit) de código abierto orientado a la seguridad en tiempo de ejecución de aplicaciones agenticas. El kit aborda riesgos emergentes (p. ej. uso indebido de herramientas, envenenamiento de memoria, escalado de privilegios) y proporciona controles, auditoría y políticas para supervisar y frenar comportamientos no deseados de agentes autónomos. Está pensado para integrarse con marcos de agentes existentes y con requisitos regulatorios próximos. Fuente: [Microsoft Open Source Blog — Agent Governance Toolkit](https://opensource.microsoft.com/blog/2026/04/02/introducing-the-agent-governance-toolkit-open-source-runtime-security-for-ai-agents/). ([opensource.microsoft.com](https://opensource.microsoft.com/blog/2026/04/02/introducing-the-agent-governance-toolkit-open-source-runtime-security-for-ai-agents/?utm_source=openai))

## Gemma 4 (Google DeepMind): modelos abiertos diseñados para ejecución local y agentes
Google/DeepMind publicó la familia Gemma 4, una nueva generación de modelos de pesos abiertos diseñada para desplegarse desde móviles hasta estaciones de trabajo. La familia incluye variantes optimizadas para edge y Mixture-of-Experts para eficiencia por token, con soporte multimodal y ventanas de contexto muy grandes en las versiones mayores. El lanzamiento enfatiza licencias permisivas (Apache 2.0) y pretende facilitar workflows agenticos y ejecución local/offline en hardware de usuario. Fuente: [Model cards — Google DeepMind (Gemma 4)](https://deepmind.google/models/model-cards/). ([deepmind.google](https://deepmind.google/models/model-cards/?utm_source=openai))

## Anthropic restringe uso de sus suscripciones con herramientas de terceros (OpenClaw y similares)
Anthropic aplicó cambios de facturación y acceso que impiden que planes de suscripción como Claude Pro/Max cubran el uso a través de ciertas herramientas/harnesses de terceros (reportes indican que OpenClaw fue afectado). El efecto inmediato obliga a muchos usuarios a migrar a facturación por uso (extra usage / pay-as-you-go) o a usar claves API directas, con impacto de costes para agentes autónomos que antes se beneficiaban de cuotas planas. Los mensajes de la comunidad y las guías de adaptación han proliferado desde el 4 de abril. Fuente: [The Next Web — Anthropic/OpenClaw (cobertura)](https://thenextweb.com/news/anthropic-openclaw-claude-subscription-ban-cost/). ([thenextweb.com](https://thenextweb.com/news/anthropic-openclaw-claude-subscription-ban-cost/?utm_source=openai))

## CVE-2026-33186: bypass de autorización en grpc-go (Go)
Se ha documentado un problema en versiones antiguas de `google.golang.org/grpc` donde un valor de `:path` en HTTP/2 sin la barra inicial puede conducir a una discrepancia entre la ruta usada por el ruteador y la expuesta a interceptores/engines de autorización, permitiendo un bypass de políticas (CVE-2026-33186). El fallo fue corregido en un PR que valida estrictamente el `:path` y rechaza métodos malformados; la recomendación inmediata es actualizar `grpc-go` a la versión parcheada y auditar interceptores que dependan de comparaciones de cadena cruda. Fuente técnica y PoC: [Code Pathfinder — análisis CVE-2026-33186].(https://codepathfinder.dev/blog/cve-2026-33186-grpc-go-authorization-bypass-malformed-path-header). ([codepathfinder.dev](https://codepathfinder.dev/blog/cve-2026-33186-grpc-go-authorization-bypass-malformed-path-header?utm_source=openai))
