## Kubernetes v1.36: resumen y novedades clave
Kubernetes v1.36 (nombre de lanzamiento "Haru") trae una oleada de graduaciones y nuevas capacidades orientadas a cargas cloud‑native y AI/ML: avances en Workload‑Aware Scheduling para tratar grupos de pods como unidades atómicas, PSI (Pressure Stall Information) graduado a GA para mejorar la observabilidad de saturación de recursos, y múltiples mejoras en Dynamic Resource Allocation (DRA) para gestionar aceleradores y recursos especializados. La release también incorpora mejoras en la API del kubelet (autorización más fina), soporte de user namespaces y novedades que facilitan ejecutar cargas sensibles a hardware (ej. VolumeGroupSnapshot). En conjunto, v1.36 sigue la tendencia de potenciar Kubernetes como plataforma para cargas AI/ML y workloads stateful a gran escala. [Fuente: notas de la release y blog oficial]. ([kubernetes.io](https://kubernetes.io/blog/2026/04/22/kubernetes-v1-36-release/?utm_source=openai))

## Deprecación y eliminación de Service ExternalIPs (Kubernetes)
El proyecto Kubernetes publicó una entrada específica sobre la deprecación y eliminación del campo `.spec.externalIPs` en Services, explicando motivos de seguridad y diseño: el campo asumía confianza total en usuarios del clúster y causó problemas operativos y de seguridad en entornos no gestionados por cloud providers. La nota incluye recomendaciones de migración para administradores (usar LoadBalancer, Gateway API o soluciones externas según el caso) y plazos de remoción en futuros lanzamientos. Es una noticia relevante para equipos que dependen de configuraciones "bare‑metal" o soluciones DIY de balanceo. [Fuente: blog oficial de Kubernetes]. ([kubernetes.io](https://kubernetes.io/blog/2026/05/14/kubernetes-v1-36-deprecation-and-removal-of-service-externalips/))

## CVE-2026-42897: vulnerabilidad en Exchange Outlook Web Access (OWA)
Microsoft divulgó el aviso relativo a CVE‑2026‑42897, que afecta componentes de Exchange Server relacionados con Outlook Web Access (OWA). La vulnerabilidad permite comportamiento de tipo XSS/spoofing en respuestas HTML de OWA; Microsoft ha publicado mitigaciones inmediatas (Exchange Emergency Mitigation Service / Exchange On‑premises Mitigation Tool) y recomienda que los administradores apliquen las mitigaciones mientras esperan parches definitivos. Los equipos de seguridad deberían revisar la guías de mitigación y duración del impacto en entornos on‑premises. [Fuente: Microsoft Tech Community y trackers de seguridad]. ([techcommunity.microsoft.com](https://techcommunity.microsoft.com/blog/exchange/addressing-exchange-server-may-2026-vulnerability-cve-2026-42897/4518498/replies/4519822?utm_source=openai))
